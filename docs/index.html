Shared Clipboard

An open-source desktop application that automatically synchronises clipboard contents between computers on the same local network. No external server, user account, or internet connection is required.

    The macOS build is produced only for Apple Silicon (M1 / M2 / M3 / M4) processors. Intel-based Macs are not currently supported.

    Overview

    The application runs quietly in the menu bar (on macOS) or the system tray (on Windows and Linux). When a copy operation is performed on one device, the content becomes instantly available on every other device on the same local network.

    Devices discover each other over mDNS (Bonjour) and communicate directly through WebSocket connections. The transmitted data does not pass through any server and never leaves the local network.

    Zero configuration

    It is sufficient for devices to be on the same Wi-Fi network; no pairing step is required.

    Local and private

    All traffic remains within the local network; no data is sent to any external service.

    Mesh topology

    Devices connect to each other directly; no central server is involved.

    System Requirements

    • macOS: version 12 (Monterey) or later on Apple Silicon.
    • Windows: Windows 10 or 11 (x64).
    • Linux: an X11 or Wayland desktop environment together with the clipboard utilities listed below.

    Installation and First Launch

    The sections below describe the security warnings and system permissions that may be encountered when the application is launched for the first time on each operating system.

    macOS — Gatekeeper warning and Local Network permission

    Because the application is not notarised under the Apple Developer Program, macOS displays a security warning on first launch. The warning can be cleared by following these steps:

    1. Open the downloaded ZIP archive and move SharedClipboard.app into the /Applications folder.
    2. Double-click the application to attempt to launch it. macOS will display a dialog titled "Apple could not verify ‘SharedClipboard’ is free of malware that may harm your Mac…" and will block the application. Dismiss the warning.
    3. Open System Settings → Privacy & Security. Scroll to the bottom of the page; an Open Anyway button is shown next to the message "SharedClipboard was blocked from use…". Click it and confirm with a password or Touch ID if prompted.
    4. In the second dialog that appears, click Open. The application will start normally from this point on.
    5. The application requests permission to discover devices on the local network. On macOS 14 and later a system dialog titled "…would like to find devices on your local network." is displayed; the Allow option must be selected. If the prompt is dismissed accidentally, the permission can be re-enabled afterwards via System Settings → Privacy & Security → Local Network.

    As an alternative, the quarantine attribute can be removed from the Terminal. When this method is used, the Open Anyway step is bypassed: 

    xattr -dr com.apple.quarantine /Applications/SharedClipboard.app
    Windows — SmartScreen and Defender Firewall

    Because the application is not signed with a code-signing certificate, Microsoft Defender SmartScreen displays a warning on first launch.

    1. Open the ZIP archive and move SharedClipboard.exe to a preferred location.
    2. When the application is launched, a blue SmartScreen window titled "Windows protected your PC" is displayed. Click More info and then Run anyway.
    3. Windows Defender Firewall raises two separate permission dialogs, one for mDNS discovery and one for the WebSocket connection. In both dialogs the Private networks checkbox should be selected and the request confirmed with Allow access.
    Linux — installing dependencies

    On Linux distributions, additional packages are required for clipboard access and the system tray. The commands below target Debian and Ubuntu based systems; equivalent package-manager commands should be used on other distributions.

    • For X11 sessions: 
      sudo apt install xclip python3-tk libappindicator3-1
    • For Wayland sessions: 
      sudo apt install wl-clipboard python3-tk
    • Mark the downloaded executable as runnable: 
      chmod +x SharedClipboard

    How It Works

    Once started, each client advertises itself over mDNS under the service type _sharedclipboard._tcp.local. and begins browsing for other devices announcing the same service type. A dedicated WebSocket connection is opened to every discovered peer, forming a full-mesh topology. When the local clipboard content changes, the new content is transmitted simultaneously to every connected peer.

    Because the architecture has no central broker, a single device going offline does not affect communication between the remaining devices. New devices are discovered automatically when they join the network, and devices that leave are removed from the active list.

    Security Notice

    Clipboard content is transmitted over the local network without encryption. The application should therefore only be used on trusted home or office Wi-Fi networks. Running it on public or unprotected wireless networks may allow other devices on the same network to read the copied data.

      macOS — Gatekeeper uyarısı ve Yerel Ağ izni

      Uygulama Apple Developer Program kapsamında notarize edilmediği için macOS, ilk açılışta bir güvenlik uyarısı gösterir. Bu uyarı şu adımlarla geçilebilir:

      1. İndirilen ZIP arşivi açılır ve SharedClipboard.app dosyası /Applications klasörüne taşınır.
      2. Uygulama çift tıklanarak başlatılmaya çalışılır. macOS, “Apple, ‘SharedClipboard’ uygulamasında kötü amaçlı yazılım olmadığını doğrulayamadı.” başlıklı bir uyarı gösterip uygulamayı engeller. Uyarı kapatılır.
      3. Sistem Ayarları → Gizlilik ve Güvenlik açılır. Sayfanın en altına doğru kaydırıldığında “SharedClipboard kullanımı engellendi…” mesajının yanında Yine de aç düğmesi görülür. Bu düğmeye basılır ve gerekirse parola ya da Touch ID ile onaylanır.
      4. Açılan ikinci dialog penceresinde Aç seçilir. Uygulama bu noktadan sonra normal şekilde başlar.
      5. Uygulama yerel ağdaki cihazları keşfetmek için izin ister. macOS 14 ve üzeri sürümlerde “…yerel ağındaki cihazları bulmak istiyor.” başlıklı sistem dialoğu çıktığında İzin Ver seçilmelidir. İzin sehven reddedildiğinde Sistem Ayarları → Gizlilik ve Güvenlik → Yerel Ağ yolu izlenerek SharedClipboard için izin daha sonra etkinleştirilebilir.

      Alternatif olarak, karantina özniteliği Terminal üzerinden de kaldırılabilir. Bu yöntem kullanıldığında yukarıdaki Yine de aç adımı atlanır: 

      xattr -dr com.apple.quarantine /Applications/SharedClipboard.app
      Windows — SmartScreen ve Defender Firewall

      3. Windows Defender Firewall, mDNS keşfi ve WebSocket bağlantısı için iki ayrı izin dialoğu açar. Her iki diyalogda da Private networks seçeneği işaretlenerek Allow access ile onaylanmalıdır.

      Nasıl Çalışır?

      Her istemci, başlatılmasının ardından kendisini _sharedclipboard._tcp.local. servis tipiyle mDNS üzerinden duyurur ve aynı tipi yayınlayan diğer cihazları aramaya başlar. Keşfedilen her cihaza ayrı bir WebSocket bağlantısı kurularak tam-örgü (mesh) bir topoloji oluşturulur. Yerel pano içeriği değiştiğinde, yeni içerik bağlı olan tüm eşlere eş zamanlı olarak iletilir.

      Mimari merkezi bir aracıya bağlı olmadığı için tek bir cihazın çevrimdışı olması diğer cihazların birbiriyle iletişimini etkilemez. Yeni cihazlar ağa katıldığında otomatik olarak keşfedilir; ağdan ayrılan cihazlar listeden düşürülür.

      Güvenlik Notu

      Pano içeriği yerel ağ üzerinde şifrelenmeden iletilir. Bu nedenle uygulama yalnızca güvenilir ev veya ofis Wi-Fi ağlarında kullanılmalıdır. Halka açık ya da parolasız kablosuz ağlarda çalıştırılması, kopyalanan verilerin aynı ağdaki başka cihazlar tarafından okunabilmesine yol açabilir.
